Video Player — your moat

Streaming is central to modern platforms; own this deeply. Master ExoPlayer / Media3, HLS vs DASH, ABR, Widevine DRM, CSAI vs SSAI, beacons, MediaSession, PiP.

Why this page matters most

Most candidates won't have hands-on player experience. You do — Bitmovin SDK, ExoPlayer/Media3, HLS, SSAI, DRM are all on your resume. The interviewer will probe; if you articulate this cleanly, your score jumps materially. If you stumble, the inflated parts of your resume become a liability. Own it.

Media3 (ExoPlayer's new home) MediaSource / TrackSelector / LoadControl HLS + CMAF adaptive bitrate (ABR) Widevine L1 / L2 / L3 SSAI vs CSAI quartile beacons (25/50/75/100) MediaSession for system integration OfflineLicenseHelper for offline DRM buffer health based switching
ExoPlayer = open-source Android media player from Google. Customizable, supports HLS/DASH/SmoothStreaming/DRM out of the box. Media3 = the modern umbrella unifying ExoPlayer + MediaSession + Cast + UI. HLS = HTTP Live Streaming (Apple), .m3u8 manifest. DASH = Dynamic Adaptive Streaming over HTTP (MPEG), .mpd manifest. ABR = Adaptive Bitrate — player picks a stream variant based on bandwidth + buffer. DRM = Digital Rights Management; Widevine is the Android standard. SSAI / CSAI = Server-/Client-Side Ad Insertion.

1. ExoPlayer / Media3 — the architecture

Media3 is the new umbrella unifying ExoPlayer + MediaSession + Cast + UI. ExoPlayer is being merged into Media3 as androidx.media3.exoplayer. New code uses Media3.

Core types (memorize these)

TypeRole
PlayerAPI surface — play, pause, seekTo, addListener. Implementation is usually ExoPlayer.
MediaSourceWhere the media comes from. HlsMediaSource, DashMediaSource, ProgressiveMediaSource, SsMediaSource (Smooth Streaming).
TrackSelectorPicks the active video/audio/text track. DefaultTrackSelector includes ABR.
LoadControlBuffering policy — min/max buffer ms, buffer-for-playback ms. DefaultLoadControl.
RendererConverts samples to output. Video/audio/text/metadata renderers.
AdsLoaderAd insertion (CSAI: IMA SDK). For SSAI, ads are baked into the stream — no AdsLoader needed.
Player.ListenerCallbacks: onPlaybackStateChanged, onPlayerError, onPositionDiscontinuity.
MediaSessionExposes playback to system (lock screen, BT, notification).

3. PlayerController — the abstraction layer (your senior signal)

Don't expose ExoPlayer directly to the rest of the app. Wrap it. Then you can swap to Bitmovin (or vice versa) by changing one impl, and tests can use a fake.

interface PlayerController {
    val state: StateFlow<PlaybackState>
    val position: StateFlow<Long>
    fun load(item: PlayerItem)
    fun play()
    fun pause()
    fun seekTo(ms: Long)
    fun release()
}

data class PlaybackState(
    val isPlaying: Boolean = false,
    val isBuffering: Boolean = false,
    val durationMs: Long = 0,
    val error: PlayerError? = null
)

class ExoPlayerController @Inject constructor(
    @ApplicationContext private val ctx: Context
) : PlayerController {
    private val _state = MutableStateFlow(PlaybackState())
    override val state = _state.asStateFlow()

    private val _position = MutableStateFlow(0L)
    override val position = _position.asStateFlow()

    private var player: ExoPlayer? = null
    private var positionJob: Job? = null

    override fun load(item: PlayerItem) {
        release()
        player = ExoPlayer.Builder(ctx).build().apply {
            addListener(listener)
            setMediaItem(MediaItem.fromUri(item.url))
            prepare()
            playWhenReady = true
        }
        positionJob = MainScope().launch {
            while (isActive) {
                _position.value = player?.currentPosition ?: 0L
                delay(500)
            }
        }
    }
    override fun play()  { player?.playWhenReady = true }
    override fun pause() { player?.playWhenReady = false }
    override fun seekTo(ms: Long) { player?.seekTo(ms) }

    override fun release() {
        positionJob?.cancel()
        player?.release()
        player = null
    }

    private val listener = object : Player.Listener {
        override fun onIsPlayingChanged(isPlaying: Boolean) {
            _state.update { it.copy(isPlaying = isPlaying) }
        }
        override fun onPlaybackStateChanged(state: Int) {
            _state.update { it.copy(isBuffering = state == Player.STATE_BUFFERING) }
        }
        override fun onPlayerError(error: PlaybackException) {
            _state.update { it.copy(error = error.toDomain()) }
        }
    }
}

Segments.ts (legacy) or .mp4 (CMAF).mp4 CMAF

Common DRMFairPlay (Apple), Widevine (CMAF)Widevine, PlayReady
Use caseCross-platform incl. iOSAndroid, web (MSE)
Modern best practice

"HLS with CMAF (.mp4 segments) + Widevine is what most major streamers ship today — a single packaging works for iOS (FairPlay on the same .mp4 segments) and Android/web (Widevine). It also lets the CDN cache once for both platforms."

5. Adaptive bitrate (ABR)

The player picks a bitrate from the manifest's variants based on:

  1. Bandwidth estimation — moving average of segment download bytes/sec.
  2. Buffer health — current buffer length vs target.
  3. Constraints — max bitrate, max resolution, data-saver toggle.

Switches up if buffer is healthy AND bandwidth allows; down if buffer drains faster than refill.

val trackSelector = DefaultTrackSelector(ctx).apply {
    parameters = buildUponParameters()
        .setMaxVideoBitrate(2_500_000)        // for data saver mode
        .setMaxVideoSize(1280, 720)            // cap to 720p on small screens
        .setForceHighestSupportedBitrate(false)
        .build()
}
val player = ExoPlayer.Builder(ctx).setTrackSelector(trackSelector).build()

CSAI (Client-Side)SSAI (Server-Side)

Buffering at breakYes (separate fetch)No (seamless)
Ad-block resistanceLow (recognizable patterns)High (looks like content)

Beacon accuracyHigh — client knows exact startNeed ad markers in stream + client beacons

Beacon tracking (you've shipped this)

Beacons are HTTP fire-and-forget pings to ad partners at lifecycle moments. Standard quartiles:

  • impression / start — playback of ad started
  • firstQuartile — 25%
  • midpoint — 50%
  • thirdQuartile — 75%
  • complete — 100%
  • click, error, skip
class BeaconTracker @Inject constructor(private val client: OkHttpClient) {
    private val fired = mutableSetOf<String>()    // dedupe per ad

    fun onProgress(adId: String, beacons: AdBeacons, currentMs: Long, durationMs: Long) {
        val pct = currentMs.toDouble() / durationMs
        when {
            pct >= 1.0 -> fire(adId, "complete", beacons.complete)
            pct >= 0.75 -> fire(adId, "thirdQuartile", beacons.thirdQuartile)
            pct >= 0.50 -> fire(adId, "midpoint", beacons.midpoint)
            pct >= 0.25 -> fire(adId, "firstQuartile", beacons.firstQuartile)
        }
    }

    private fun fire(adId: String, event: String, urls: List<String>) {
        val key = "$adId:$event"
        if (!fired.add(key)) return       // already fired
        urls.forEach { url ->
            client.newCall(Request.Builder().url(url).build())
                .enqueue(object : Callback {
                    override fun onFailure(call: Call, e: IOException) { /* log */ }
                    override fun onResponse(call: Call, response: Response) { response.close() }
                })
        }
    }
}

8. MediaSession (system integration)

Exposes playback state to the system: lock screen controls, Bluetooth headset buttons, Wear OS, Auto, Cast.

class PlaybackService : MediaSessionService() {
    private lateinit var session: MediaSession

    override fun onCreate() {
        super.onCreate()
        val player = ExoPlayer.Builder(this).build()
        session = MediaSession.Builder(this, player)
            .setCallback(MediaSessionCallback())
            .build()
    }
    override fun onGetSession(info: MediaSession.ControllerInfo): MediaSession = session
    override fun onDestroy() { session.release(); super.onDestroy() }
}

10. Compose integration

@Composable
fun PlayerScreen(player: ExoPlayer, modifier: Modifier = Modifier) {
    val lifecycleOwner = LocalLifecycleOwner.current
    DisposableEffect(player, lifecycleOwner) {
        val observer = LifecycleEventObserver { _, event ->
            when (event) {
                Lifecycle.Event.ON_PAUSE -> player.pause()
                Lifecycle.Event.ON_RESUME -> player.play()
                else -> Unit
            }
        }
        lifecycleOwner.lifecycle.addObserver(observer)
        onDispose { lifecycleOwner.lifecycle.removeObserver(observer) }
    }

    AndroidView(
        factory = { ctx ->
            PlayerView(ctx).apply {
                this.player = player
                useController = true
            }
        },
        update = { it.player = player },
        modifier = modifier.fillMaxWidth().aspectRatio(16f / 9f)
    )

    DisposableEffect(Unit) {
        onDispose {
            // If you OWN the player at this scope, release here:
            // player.release()
        }
    }
}

11. Quality / observability

Track these metrics in production (Conviva, Bitmovin Analytics, or your own pipeline):

  • TTFF — Time To First Frame.
  • Rebuffer ratio — rebuffer time / total play time.
  • Bitrate distribution — what % of playtime at each variant.
  • Errors — DRM errors, network errors, decoder errors, fatal vs recoverable.
  • Concurrency — average concurrent streams per user (DRM seat concerns).

12. Common player interview probes

Q. Why ExoPlayer over MediaPlayer?

MediaPlayer is OS-tied — limited customization, no full ABR, DRM only via OS support, hard to debug, behavior varies by OEM. ExoPlayer is open-source, configurable, consistent across OS versions, supports HLS/DASH/SmoothStreaming/DRM out of the box.

Q. How do you handle a 4G→Wifi switch mid-playback?

ExoPlayer recovers automatically — segments fail, retry, continue. If retries exhaust, surface a recoverable error and call player.prepare() again. The DASH/HLS adaptive bitrate logic re-evaluates bandwidth on the new connection.

Q. How do you A/B test player builds?

Behind a feature flag, swap the PlayerController implementation (e.g., Bitmovin vs ExoPlayer for one cohort). Compare TTFF, rebuffer ratio, error rate. Roll forward or back based on metrics.

Q. How do you do offline playback?

(1) Download stream segments via DownloadManager. (2) Pre-fetch and store the Widevine license via OfflineLicenseHelper. (3) On playback, hand the license keyId to the player via DrmConfiguration.setKeySetId().

Q. What's the difference between SSAI and CSAI?

CSAI: client fetches ad and pauses content during ad break (separate fetch, vulnerable to ad blockers, easier per-user personalization). SSAI: server stitches ads into the master stream — single continuous playback, no buffer at break, harder to ad-block, harder per-user personalization (needs per-user manifests).

Q. How do you know an ad's quartile beacons fired?

Track beacon-fire events client-side and send to your analytics. Compare ad partner reports vs your client beacons — discrepancies (>2-3%) indicate beacon delivery issues, partner-side processing lag, or ad-blocker activity.

Q. How do you handle DRM L3 → L1 fallback?

Capability-detect at startup: query MediaDrm for the supported security level. If L1 unavailable but content requires it, refuse playback or downgrade to lower-tier content. Don't silently serve SD on a 4K device — confuses users.
