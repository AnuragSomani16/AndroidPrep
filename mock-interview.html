Mock Interview — 200+ Q&A

Filter by difficulty. Attempt the answer aloud first, then expand. Mark questions you stumbled on; redo them tomorrow.

Difficulty:

Kotlin (1–35)

Basic Q1. Difference between val and var?

val is read-only after initialization (immutable reference; the underlying object can still be mutable). var can be reassigned. Use val by default.

Basic Q2. What is null safety in Kotlin?

The type system distinguishes nullable (String?) from non-null (String). The compiler enforces null checks before access. Operators: ?. (safe call), ?: (Elvis), !! (force non-null — avoid). Compile-time NPE protection.

Basic Q3. What is a data class?

Auto-generates equals, hashCode, toString, copy, componentN. Must have ≥1 constructor parameter. Can't be abstract, open, sealed, inner. Non-constructor var fields are NOT in equals/hashCode/copy — common bug source.

Basic Q4. What does !! do?

Asserts the value is non-null; throws NullPointerException if it isn't. Acceptable in tests; a code smell in production. Use ?.let, ?:, or checkNotNull instead.

Basic Q5. What's a sealed class / sealed interface?

A type whose subclasses are all known at compile time → exhaustive when. Use for closed hierarchies like UiState.Loading/Success/Error. Sealed interface (1.5+) allows multiple-inheritance composition.

Basic Q6. What does const val mean?

Compile-time constant. Only at top level / object / companion object. Only primitives + String. Inlined at the call site; saves a getter call vs val.

Basic Q7. Differences between String concatenation, StringBuilder, and String templates in Kotlin?

String templates ("Hello $name") compile to StringBuilder.append chains — efficient for typical use. For tight loops with many appends, instantiate a StringBuilder yourself. Plain + on String creates intermediate Strings — fine for occasional use, slow in loops.

Basic Q8. What is Any vs Object?

Any is Kotlin's root class (analogous to Java's Object). On the JVM, Kotlin's Any compiles to java.lang.Object. Differences: Kotlin's Any has only equals, hashCode, toString — no wait/notify/getClass.

Basic Q9. List vs MutableList?

List<T> is read-only (no add/remove methods). MutableList<T> adds them. Both are interfaces; underlying implementation is typically ArrayList. Read-only ≠ immutable — another reference may mutate the same list.

Basic Q10. What does fun foo(x: Int = 5, y: String = "hi") demonstrate?

Default arguments. Combined with named-args (foo(y = "world")) eliminates most builder-pattern boilerplate.

Intermediate Q11. lateinit vs by lazy?

lateinit var: only var, only non-null reference types, must be initialized before use. by lazy: only val, thread-safe (SYNCHRONIZED) by default, computed on first access. Use lateinit for DI/view binding; by lazy for expensive deferred init.

Intermediate Q12. Explain scope functions (let/run/with/apply/also).

let: it / lambda result — nullable transform. run: this / lambda result — block on receiver. with: this / lambda result — group calls. apply: this / receiver — builder. also: it / receiver — side effect. Mnemonic: apply/also return the receiver; others return the lambda result.

Intermediate Q13. == vs ===?

== calls equals() — structural equality. === is referential identity — same object. For nullables, both correctly handle null comparisons.

Intermediate Q14. What's an extension function and how is it dispatched?

A function defined outside a class but called as if it were a member: fun String.isEmail(): Boolean = .... Statically dispatched — resolved at compile time based on the static (declared) type, not the runtime type. A member with the same signature wins over an extension.

Intermediate Q15. What is an inline function and when to use it?

Compiler inlines the function body at each call site. Avoids lambda object allocation; supports non-local return. Use for high-frequency higher-order functions (forEach, let). Don't inline large functions — bytecode bloat.

Intermediate Q16. What is reified and why does it require inline?

reified makes the type parameter accessible at runtime as T::class. Only works inside inline functions because inline copies the body — including the type parameter — to the call site, where T is concrete. Used in APIs like gson.fromJson<T>(json) or intent.getParcelable<T>(key).

Intermediate Q17. Explain variance: out, in, invariant.

out T (covariant): producer of T; Box<Dog> is-a Box<Animal>. T can only appear in OUT positions. in T (contravariant): consumer; Comparator<Animal> is-a Comparator<Dog>. T can only appear in IN positions. Invariant (default): no subtype relationship — like MutableList<T>.

Intermediate Q18. What's the difference between object and companion object?

object = singleton. Lazily initialized, thread-safe. companion object inside a class = singleton tied to the class (Java's static analog). Use @JvmStatic on companion members for true Java-static interop.

Intermediate Q19. What is class delegation (by)?

Forwards interface methods to a constructor argument: class Cached(impl: Repo) : Repo by impl { /* override only what you need */ }. Composition over inheritance with zero boilerplate.

Intermediate Q20. Property delegation — what's by lazy doing under the hood?

val x by lazy { ... } compiles to a backing Lazy<T> property whose getValue returns the cached value (computed once). Other built-in delegates: Delegates.observable, Delegates.vetoable, Delegates.notNull, map delegation.

Intermediate Q21. Sealed class vs enum — when which?

Enum: fixed instances with the same fields/methods (Direction.NORTH/SOUTH/EAST/WEST). Sealed: hierarchy where each subclass can have unique fields (UiState.Loading, UiState.Success(items), UiState.Error(throwable)). Use sealed for state with associated data; enum for plain enumerations.

Intermediate Q22. What's a value class and when to use it?

@JvmInline value class UserId(val raw: String) — wraps a single value with no runtime overhead in most cases (no boxing on direct calls). Use for type-safe primitives — fetch(userId, email) can't be called with the args swapped at compile time.

Intermediate Q23. What is operator overloading? Give a senior-level use.

operator fun on standard names (plus, minus, get, set, invoke, etc.). Senior pattern: operator fun invoke for use cases — class GetUser @Inject constructor(...) { suspend operator fun invoke(id) = ... }; called as getUser(id).

Advanced Q24. crossinline vs noinline?

Both modify inline function lambda parameters. crossinline: forbids non-local returns (used when the lambda is called from a different execution context, e.g., stored in another lambda or thread). noinline: opt out of inlining for that one parameter (when you need to pass the lambda as an object, e.g., to another non-inline function).

Advanced Q25. Why does data class with a var body field break equals?

Data class generates equals/hashCode/copy from constructor parameters only. A var declared in the body is NOT in those — so two instances differing only in body fields are still ==. Always put fields in the constructor.

Advanced Q26. What's typealias good for?

Aliases for complex types: typealias OnUserSelected = (User) -> Unit. Improves readability without creating a distinct type. (For type safety distinctness, use value class instead.)

Advanced Q27. What's a contract in Kotlin (e.g., contract { returns() implies (x is Int) })?

Compiler hints describing the function's behavior to enable smarter smart-casts. E.g., contract { callsInPlace(block, EXACTLY_ONCE) } tells the compiler the lambda runs exactly once, enabling val-init-after-call. Used by stdlib (let, also, require, requireNotNull).

Advanced Q28. Sequence vs Iterable — when which?

Iterable: each operator allocates a new collection; eager. Sequence: lazy, terminal-driven; better for big collections with many ops; worse for small ones (per-element overhead). Convert with list.asSequence()...toList().

Advanced Q29. Spot the bug: 
data class User(val id: String) { var lastSeen: Long = 0 }
val a = User("1"); a.lastSeen = 100
val b = User("1"); b.lastSeen = 200
println(a == b)

Prints true. lastSeen is a body var — NOT in generated equals/hashCode/copy. Either move it to the constructor or use a non-data class.

Advanced Q30. Spot the bug: 
open class A { open fun hi() = "A" }
class B : A() { override fun hi() = "B" }
fun A.greet() = "Hi A"
fun B.greet() = "Hi B"
val a: A = B()
println(a.greet())

Prints "Hi A". Member functions are dynamically dispatched; extension functions are statically dispatched based on the declared type. a is declared A, so A.greet() wins.

Advanced Q31. Why does this code not compile? 
val box: Box<Animal> = Box<Dog>(Dog())
class Box<T>(val item: T)

Generic types are invariant by default — Box<Dog> is NOT a subtype of Box<Animal>. Make T covariant: class Box<out T>(val item: T) — works because T only appears in OUT positions.

Advanced Q32. What's the difference between List<Any> and List<*>?

List<Any>: list of Any; you can read items as Any or write Any values. List<*>: star projection — you can read but the element type is Any?; you can't write because the element type is unknown. Use * when you only consume from a list of unknown T.

Extreme Q33. What does Nothing mean and when do you encounter it?

Nothing is a subtype of every type — used for "this never returns" (functions that throw or loop forever): throw IllegalStateException(), error("..."), TODO(). Also enables covariance: sealed class Result<out T> { class Err : Result<Nothing>() } — Err can be a Result of any T.

Extreme Q34. What's the bytecode-level difference between a Kotlin data class and a Java POJO?

Data class generates equals, hashCode, toString, component1..N, copy methods automatically. The class itself is final by default. Properties are exposed as private fields + getters (and setters for var). At the JVM level, the result is a typical immutable POJO — usable from Java but with extra componentN methods.

Extreme Q35. Spot the issue: 
inline fun foo(crossinline cb: () -> Unit) {
    val r = Runnable { cb() }
    Thread(r).start()
}
foo { return }   // ?

Compile error. crossinline forbids non-local returns in the lambda — and the lambda is being captured into Runnable, which the compiler can't inline. The non-local return would jump out of the calling function from a different thread, which is impossible. crossinline exists exactly to forbid this.

Coroutines & Flows (36–80)

Basic Q36. What is a coroutine?

A suspendable computation. Cheaper than a thread (thousands at once); runs on a thread (managed by a Dispatcher) and can suspend at suspend points without blocking the thread.

Basic Q37. launch vs async?

Both start a coroutine. launch returns Job — fire-and-forget; uncaught exception propagates immediately. async returns Deferred<T> — call await() to get result; exceptions held until await.

Basic Q38. Name the standard Dispatchers and their use cases.

Main: UI thread updates. Default: CPU-bound (parsing, sorting). IO: blocking I/O (network, disk). Unconfined: rarely used — resumes on whichever thread triggered resumption.

Basic Q39. What is viewModelScope?

A pre-built CoroutineScope tied to a ViewModel's lifecycle. Cancelled in onCleared(). Use it for VM business logic.

Basic Q40. What does suspend do?

Marks a function as suspendable. The compiler transforms it into a state machine that can pause/resume without blocking the underlying thread. Can only be called from another suspend function or a coroutine builder.

Intermediate Q41. Explain structured concurrency.

Coroutines are tied to a CoroutineScope. Children depend on the parent's lifecycle. Parent cancel → children cancel. Child failure → parent fails (unless SupervisorJob). A coroutine can never outlive its scope. Eliminates GlobalScope-style leaks.

Intermediate Q42. What does SupervisorJob do?

A Job where child failures don't cancel siblings or the parent. Used inside supervisorScope { } or as a custom scope's context: CoroutineScope(SupervisorJob() + Dispatchers.Default).

Intermediate Q43. coroutineScope { } vs supervisorScope { }?

coroutineScope: any child failure cancels all siblings and propagates up. supervisorScope: children are independent — one's failure doesn't kill others. coroutineScope for atomic groups; supervisorScope for resilient parallel fetches.

Intermediate Q44. How does cancellation work in coroutines?

Cooperative — cancel() sets a flag; suspending functions check it (most stdlib ones do). For CPU loops, call ensureActive() or yield() periodically. finally blocks run; for cleanup that must suspend, wrap in withContext(NonCancellable).

Intermediate Q45. How do you handle exceptions in launch vs async?

launch: uncaught exceptions go to the parent / CoroutineExceptionHandler. async: exception is held in Deferred until await() — so wrap await() in try/catch. CoroutineExceptionHandler does NOT catch async exceptions until awaited.

Intermediate Q46. What's withContext?

Switches dispatcher inside an existing coroutine. Doesn't start a new coroutine. Returns when the block completes. Use for "do this part on IO, the rest on Main": val data = withContext(Dispatchers.IO) { api.fetch() }.

Intermediate Q47. withTimeout vs withTimeoutOrNull?

withTimeout: throws TimeoutCancellationException. withTimeoutOrNull: returns null on timeout. Use the null variant when you can fall back gracefully.

Intermediate Q48. Cold vs hot Flow?

Cold (Flow): producer block runs per collector; no collector → no work. Hot (StateFlow, SharedFlow): emits independently of collectors; collectors see whatever is currently being emitted (or replay).

Intermediate Q49. StateFlow vs SharedFlow?

StateFlow: always has a value, conflated, replay = 1, equality dedupe — for state. SharedFlow: configurable replay/buffer, no dedupe — for events. StateFlow is essentially SharedFlow(replay=1, BufferOverflow.DROP_OLDEST) + conflation + equality dedupe.

Intermediate Q50. What is flatMapLatest?

For each upstream emission, cancels the previous inner Flow and starts a new one. Search-as-you-type pattern: each new query cancels the in-flight search.

Intermediate Q51. What does stateIn(scope, SharingStarted.WhileSubscribed(5000), initial) do?

Converts a cold Flow to a hot StateFlow. The 5000ms timeout keeps upstream alive after the last collector unsubscribes — handles configuration changes (rotation) without re-fetching. The standard pattern.

Intermediate Q52. combine vs zip vs merge?

combine: emits whenever any source emits, using latest from each. zip: pairs up; waits for both; emits when both have a new value. merge: interleaves emissions.

Intermediate Q53. What is flowOn?

Sets the dispatcher for upstream operators. Doesn't affect downstream. Used to shift the producer's work off main: flow { ... }.map { ... }.flowOn(IO).collect { ... } — the flow + map run on IO, collect runs on caller dispatcher.

Intermediate Q54. collectAsState vs collectAsStateWithLifecycle?

collectAsState: starts collecting and never stops while in composition (background work even when off-screen). collectAsStateWithLifecycle: stops when the lifecycle drops below STARTED. Always use the lifecycle-aware variant.

Advanced Q55. What's repeatOnLifecycle(Lifecycle.State.STARTED) { }?

For Activity/Fragment Flow collection. Restarts the block each time the lifecycle reaches STARTED, cancels when it drops below. Equivalent to collectAsStateWithLifecycle in Compose.

Advanced Q56. Why doesn't my SharedFlow deliver events when there's no collector?

Default: replay=0, extraBufferCapacity=0. emit() suspends; tryEmit() drops. For events: MutableSharedFlow(replay=0, extraBufferCapacity=1, onBufferOverflow=DROP_OLDEST) — buffer 1 in case the collector is briefly absent.

Advanced Q57. Why doesn't my StateFlow update sometimes?

StateFlow uses structural equality dedupe. If newValue.equals(oldValue), no emission. Use update { it.copy(...) } with actual changes, and ensure your data class's equals reflects the changing fields.

Advanced Q58. How do you bridge a callback API to Flow?

callbackFlow { register listener that calls trySend; awaitClose { unregister } }. awaitClose is required so the producer suspends until the collector cancels.

Advanced Q59. buffer vs conflate vs collectLatest?

All address backpressure. buffer(n): queue up to n items between producer and consumer. conflate: drop intermediate; consumer always sees latest. collectLatest: cancel in-progress collector when a new item arrives.

Advanced Q60. Why do try { ... } catch (e: Exception) { ... } blocks "swallow" cancellation?

CancellationException is an Exception — gets caught silently, breaking cooperative cancellation. Always rethrow it: if (e is CancellationException) throw e. Or catch only the specific exceptions you handle.

Advanced Q61. How do you make a CPU-heavy loop cancellable?

Call ensureActive() or yield() periodically inside the loop. Suspending functions check cancellation automatically; pure CPU loops do not — you'd otherwise see "cancelled" coroutines still running.

Advanced Q62. What's NonCancellable and when to use it?

A Job in the context that ignores cancellation. Use inside finally blocks for cleanup that must suspend: finally { withContext(NonCancellable) { release() } }.

Advanced Q63. Walk through a search-as-you-type pipeline. 
val results = queryFlow
    .debounce(300)
    .distinctUntilChanged()
    .flatMapLatest { q -> if (q.isBlank()) flowOf(emptyList()) else repo.search(q) }
    .stateIn(viewModelScope, SharingStarted.WhileSubscribed(5000), emptyList())

Debounce drops bursts; distinctUntilChanged dedupes consecutive same query; flatMapLatest cancels in-flight when new query arrives; stateIn turns it hot with config-change tolerance.

Advanced Q64. How do you test coroutine code?

runTest { } with virtual time. Inject your dispatcher (StandardTestDispatcher) — never hardcode Dispatchers.IO. Use advanceTimeBy, advanceUntilIdle. Dispatchers.setMain(testDispatcher) for VM tests. Turbine for Flow assertions.

Advanced Q65. What are Channel capacities?

Channel.RENDEZVOUS (0): send suspends until receive. Channel.UNLIMITED: unbounded. Channel.CONFLATED: keep only the latest. Channel.BUFFERED: default ~64. Or specify an int.

Extreme Q66. Trace this: 
scope.launch {
    val a = async { fetchA() }   // throws
    val b = async { fetchB() }
    a.await()  // ?
    b.await()
}

If scope is coroutineScope (no SupervisorJob), fetchA's exception cancels b too, propagates to the outer launch. With SupervisorJob, b survives; a.await() rethrows; if uncaught, the outer launch fails.

Extreme Q67. Why is GlobalScope.launch almost always wrong?

Bypasses structured concurrency — the coroutine outlives any meaningful lifetime, leaks references, prevents tests from waiting for it cleanly. The only legitimate use is at the very top of Application.onCreate for app-scoped work — and even then, prefer CoroutineScope(SupervisorJob() + Dispatchers.Default) stored as a property so you can cancel during tests.

Extreme Q68. How does kotlinx.coroutines implement suspension under the hood?

The compiler transforms each suspend function into a state machine. At each suspension point, it saves the local state into a Continuation object and returns. When resumed, it picks up where it left off. The dispatcher schedules continuations on threads. No magic — just CPS (continuation-passing style) generated by the compiler.

Extreme Q69. How would you implement parallel mapping with bounded concurrency? 
suspend fun <T, R> List<T>.parallelMap(parallelism: Int = 4, transform: suspend (T) -> R): List<R> =
    coroutineScope {
        val sem = Semaphore(parallelism)
        map { item -> async { sem.withPermit { transform(item) } } }.awaitAll()
    }
Extreme Q70. Mutex vs synchronized?

synchronized blocks the underlying thread — bad for coroutines (could deadlock with single-threaded dispatchers like Main). Mutex.withLock is suspendable — yields the thread while waiting. Always use Mutex in coroutine contexts.

Intermediate Q71. What's the difference between distinctUntilChanged and a regular distinct?

distinctUntilChanged dedupes consecutive equals — A B A is preserved as A B A. There's no general distinct on Flow because true distinct would require remembering all past values.

Intermediate Q72. onEach vs map?

onEach { }: side effect per emission; passes the value through unchanged. map { }: transforms each emission. Use onEach for logging or analytics in a chain.

Intermediate Q73. SharingStarted.Eagerly vs Lazily vs WhileSubscribed?

Eagerly: starts immediately when scope starts. Lazily: starts on first collector, never stops. WhileSubscribed(timeout): starts on first collector, stops after timeout post-last-unsubscribe. WhileSubscribed(5000) is the standard.

Intermediate Q74. What's the relationship between Job, SupervisorJob, and CoroutineScope?

A CoroutineScope's context contains a Job (or SupervisorJob). Children of coroutines launched in this scope have this Job as parent. Cancelling the parent Job cancels the scope. SupervisorJob differs in that one child's failure doesn't propagate to siblings.

Advanced Q75. Why is exception handling different in launch vs async?

launch is "fire and forget" — exceptions need somewhere to go (handler, parent). async represents a future result; the exception is part of that result, surfaced when you call await(). If you never await, the exception is silently held (and propagates via parent in non-supervisor scopes).

Advanced Q76. How do you handle a 401 response when multiple parallel requests hit it simultaneously?

OkHttp's Authenticator with a Mutex.withLock around the refresh call. First 401 acquires the lock, refreshes, releases. Subsequent 401s see the new token (compare-and-swap pattern: check current vs expired token, skip refresh if already updated).

Advanced Q77. What happens when you collect a Flow on the wrong dispatcher?

If your producer blocks (e.g., DB read), collecting on Main blocks the UI. Use flowOn(IO) upstream to shift the producer; collect on Main for UI updates.

Advanced Q78. Difference between shareIn and stateIn?

Both convert cold→hot. stateIn: returns StateFlow with conflation, equality dedupe, replay=1, requires initial value. shareIn: returns SharedFlow with configurable replay/buffer, no dedupe, no initial value. Use stateIn for screen state; shareIn for event streams.

Extreme Q79. Build a rate-limited Flow that emits at most 1 item per second. 
fun <T> Flow<T>.throttleFirst(periodMs: Long): Flow<T> = flow {
    var lastEmit = 0L
    collect {
        val now = System.currentTimeMillis()
        if (now - lastEmit >= periodMs) {
            lastEmit = now
            emit(it)
        }
    }
}
Extreme Q80. Why is GlobalScope.launch { /* ... */ } dangerous in Compose?

Compose composables are recomposed many times. If you start a coroutine in GlobalScope from a composable body, you'll spawn one per recomposition — leak. Use LaunchedEffect (key-tied) or rememberCoroutineScope() (event-triggered).

Jetpack Compose (81–115)

Basic Q81. What is Jetpack Compose?

Android's declarative UI toolkit. You describe what UI should look like for a given state via @Composable functions; the runtime diffs and updates underlying nodes. Replaces XML layouts and View hierarchies for new code.

Basic Q82. Why Compose over XML?

State-driven (no findViewById, no manual setText), composable functions are testable as plain Kotlin, less boilerplate, first-class animations via animate*AsState, type-safe theming, incremental migration via ComposeView / AndroidView.

Basic Q83. What is recomposition?

Re-running parts of the composition tree when state inputs change. Compose uses a slot table for positional memoization to know what to re-run vs reuse.

Basic Q84. What is state hoisting?

Lifting state to the lowest common parent and passing state + onEvent down to stateless composables. Makes children easier to preview, test, and reuse.

Basic Q85. remember vs rememberSaveable?

remember: survives recomposition; lost on configuration change. rememberSaveable: also survives configuration change AND process death (Bundle-based).

Intermediate Q86. What are stable types in Compose?

Types Compose can trust to mean "if equals returns true, the rendered output is the same." Primitives, String, lambdas, types annotated @Stable/@Immutable, data classes with all-stable val fields. Lists/Maps are unstable. Stable parameters enable recomposition skipping.

Intermediate Q87. LaunchedEffect vs SideEffect vs DisposableEffect?

LaunchedEffect(key): launch a coroutine; restarts on key change. SideEffect: runs after every successful composition; sync state to non-Compose code. DisposableEffect(key): setup + onDispose { cleanup }.

Intermediate Q88. What is derivedStateOf?

Wraps a state read so dependents recompose only when the derived value changes — not when underlying state ticks. Classic use: val showFab by remember { derivedStateOf { listState.firstVisibleItemIndex == 0 } }.

Intermediate Q89. What is rememberUpdatedState for?

Fixing the stale-closure problem in long-running effects. val current by rememberUpdatedState(callback) always points to the latest value — so a LaunchedEffect(Unit) doesn't capture an old version of the lambda.

Intermediate Q90. What is CompositionLocal?

Implicitly passes values down the composition tree without prop drilling. Use for cross-cutting concerns (theme, density, snackbar host). compositionLocalOf tracks reads (only readers recompose); staticCompositionLocalOf doesn't track (all descendants recompose on change — cheaper for rarely-changing values).

Intermediate Q91. Why does Modifier order matter?

Modifiers compose left-to-right, outside-to-inside. padding(16).background(Red) = padding around red bg. background(Red).padding(16) = red bg with padding inside (smaller red).

Intermediate Q92. Why must LazyColumn items have a key?

Without key, Compose treats items by position — reordering causes wrong recomposition, scroll jumps, and animations targeting wrong items. key = { it.id } always.

Advanced Q93. How do you debug "my composable recomposes too much"?

(1) Layout Inspector → "Show recomposition counts." (2) Compose Compiler reports — shows per-function stability/skippability. (3) Look for unstable parameters — Lists, Maps, var fields, lambdas allocated per composition. Fix with @Immutable, ImmutableList, or remember { { ... } }.

Advanced Q94. How do you persist UI state across process death in Compose?

rememberSaveable for trivial UI state (Bundle-based; small data only). For ViewModel-owned state, inject SavedStateHandle and use savedState.getStateFlow(key, default). For larger state, persist to Room or DataStore.

Advanced Q95. How does snapshotFlow work?

Wraps a State-reading lambda into a Flow. Compose's snapshot system detects which State objects the lambda reads; the Flow re-emits when any of them change. Use to bridge Compose State to non-Compose code: snapshotFlow { listState.firstVisibleItemIndex }.collect { ... }.

Advanced Q96. What's the difference between Modifier.composed and a regular Modifier function?

Regular Modifier extension: stateless. Modifier.composed { }: runs in a composition scope, can call composables, can remember state. Allocates per composition — use sparingly. For most cases, prefer plain Modifier extensions or wrap state externally.

Advanced Q97. How would you implement a custom Modifier that scales the child on press? 
fun Modifier.scaleOnPress() = composed {
    var pressed by remember { mutableStateOf(false) }
    val scale by animateFloatAsState(if (pressed) 0.95f else 1f, label = "scale")
    pointerInput(Unit) {
        awaitEachGesture {
            awaitFirstDown(); pressed = true
            waitForUpOrCancellation(); pressed = false
        }
    }.graphicsLayer { scaleX = scale; scaleY = scale }
}
Advanced Q98. What's the diff between compositionLocalOf and staticCompositionLocalOf?

compositionLocalOf: tracks reads — only composables that actually read the value recompose when it changes. staticCompositionLocalOf: doesn't track — when value changes, ALL descendants recompose. Use static for rarely-changing values (theme); dynamic for values that change often.

Advanced Q99. How do you avoid the "lambda allocated per composition" gotcha?

For function-references (vm::onClick): they're stable, no fix needed. For inline lambdas passed to children: extract to remember { { } }, OR rely on Compose Compiler's stability — function types are stable starting from a recent version. Function reference is the cleanest.

Advanced Q100. What's the role of the slot table?

The runtime data structure holding composition state. Each composable call has a "slot" identified by call-site location. Compose uses positional memoization — at the same position, remember returns the same value across recompositions. Reordering or conditional inclusion of composables changes positions.

Extreme Q101. Why does if (condition) { remember { 0 } } else { remember { 0 } } behave differently than remember { if (condition) 0 else 0 }?

Different slot positions. The first form has two remember calls — one inside each branch. Toggling condition swaps which slot is active, losing the other's state. The second form has one remember call at a stable position. Lesson: structure remember calls so position is stable across compositions.

Extreme Q102. How would you debug a Compose performance regression?

(1) Reproduce in profiler with System Trace to see frame timings. (2) Layout Inspector recomposition counts on the suspect screens. (3) Compose Compiler reports — diff against the last good commit. (4) Macrobenchmark for objective numbers. (5) Check baseline profiles still cover the changed code.

Extreme Q103. How do you implement Pull-to-Refresh in Compose? 
val refreshState = rememberPullToRefreshState()
LaunchedEffect(refreshState.isRefreshing) {
    if (refreshState.isRefreshing) {
        vm.refresh()                  // VM updates a Flow when done
        refreshState.endRefresh()
    }
}
Box(Modifier.nestedScroll(refreshState.nestedScrollConnection)) {
    LazyColumn { ... }
    PullToRefreshContainer(refreshState, Modifier.align(Alignment.TopCenter))
}
Intermediate Q104. How do you handle navigation in Compose?

Navigation Compose: NavHost(navController, startDestination) + composable<Route> { }. For type-safe routes (Nav 2.8+), use @Serializable data classes. navController.navigate(Route(id)) to push; navController.popBackStack() to go back.

Intermediate Q105. Why use Modifier.testTag?

For UI testing — finds the node via composeTestRule.onNodeWithTag("X"). For accessibility, use Modifier.semantics { contentDescription = "..." } or Modifier.contentDescription("...").

Intermediate Q106. What's produceState?

Turns a non-Compose source into a State<T>. Provides a coroutine scope and an awaitDispose for cleanup: val state by produceState(initial = "...") { value = repo.fetch(); awaitDispose { } }.

Intermediate Q107. animateFloatAsState vs updateTransition?

animateFloatAsState: animate one value. updateTransition: orchestrate multiple animations driven by the same state — alpha, color, size all animating together when state changes.

Intermediate Q108. How do you do dark theme in Compose?

val colorScheme = if (isSystemInDarkTheme()) darkColorScheme() else lightColorScheme(); pass to MaterialTheme(colorScheme = colorScheme) { content() }. Override per-app by reading user preference from DataStore.

Advanced Q109. Why do I get "@Composable invocations can only happen from the context of a @Composable function"?

You're calling a composable from a non-composable lambda — e.g., from onClick. Composables can only be called inside other composables. Use rememberCoroutineScope() or hoist the action.

Advanced Q110. What's the lifecycle of LaunchedEffect?

Starts a coroutine when the effect enters the composition. Cancels and restarts when the key changes. Cancels permanently when the composable leaves the composition.

Advanced Q111. How do you embed a legacy XML View in Compose?

AndroidView(factory = { ctx -> LegacyView(ctx) }, update = { it.refresh(state) }). The update block runs on each recomposition.

Advanced Q112. How do you embed a Composable in an XML layout?

<androidx.compose.ui.platform.ComposeView> in XML; in code, binding.composeView.setContent { MyComposable() }.

Advanced Q113. Why does my LaunchedEffect(Unit) still re-run on rotation?

It shouldn't if the composable is preserved through configuration change — but if you're recreating the Activity with setContent in onCreate, every config change re-runs onCreate → recomposes everything → LaunchedEffect(Unit) restarts. The fix: hold state in a ViewModel; the composable reads state from there and the effect doesn't need to re-fire.

Extreme Q114. How would you implement a Compose-friendly bottom sheet that supports drag-to-dismiss?

Use Material3's ModalBottomSheet with rememberModalBottomSheetState. For full custom: AnchoredDraggable, swipeable, or build with Modifier.draggable + your own animation logic. Senior take: prefer the library — anchored draggable is mature and accessibility-friendly.

Extreme Q115. How would you implement a sticky header in LazyColumn?

stickyHeader { } within a LazyColumn. Each stickyHeader is pinned to the top while its section is visible. Note: stickyHeader is on LazyListScope, available in LazyColumn (and LazyRow with caveats).

Architecture (116–135)

Basic Q116. MVVM vs MVI in one sentence each?

MVVM: ViewModel exposes state via observable streams; view observes; events flow back via methods. MVI: strict UDF — Intent → Reducer → State → View, with side effects on a separate channel.

Basic Q117. What's a Repository?

The single source of truth for a data type. Coordinates between data sources (network, DB, in-memory cache) and exposes domain models (not entities/DTOs).

Basic Q118. What is Clean Architecture?

A layering approach: presentation depends on domain; data depends on domain. Domain has no Android dependency. Inner layers know nothing about outer (the Dependency Rule).

Intermediate Q119. Why use Use Cases?

Encapsulate business logic that's reused across screens, composes multiple repos, or has non-trivial logic worth unit-testing. Skip them when the VM just calls a single repo method passthrough.

Intermediate Q120. When would you choose MVI over MVVM?

State-heavy screens (player, multi-step forms, search, complex animation states). When "what's the screen showing right now?" becomes hard to answer with separate flows. The single-state-object discipline pays off there.

Intermediate Q121. What's the offline-first pattern?

DB is the source of truth for the UI. UI observes dao.observeAll() as a Flow. Repository's refresh() writes through DB → UI auto-updates. WorkManager handles background sync. This guarantees no UI/cache drift.

Intermediate Q122. Why multi-module?

Build speed (parallel compilation, unchanged modules cached), enforced layer boundaries (the build won't let UI call into data layer if you didn't declare the dep), explicit API surface, reusability across app variants.

Intermediate Q123. api vs implementation dependency in Gradle?

implementation: dep is internal — consumers of this module don't see it. api: dep is exposed — consumers see it transitively. Default to implementation; use api only when types from the dep appear in your module's public API.

Intermediate Q124. What's "dependency inversion" in mobile architecture?

Define interfaces in the inner (domain) layer; implementations live in outer (data) layer. The data layer depends inward on domain, not the other way. Lets you swap implementations and unit-test domain without Android.

Advanced Q125. How do you avoid circular module dependencies?

Define interfaces in the inner-most module that needs them; have outer modules implement. Use a :core:common module for shared types. Feature modules depend only on :domain, :design-system, :common — never on each other.

Advanced Q126. How do you handle one-shot events in MVI without losing them on rotation?

SharedFlow(replay=0, extraBufferCapacity=1, BufferOverflow.DROP_OLDEST). Collected from a LaunchedEffect(Unit) in Compose or repeatOnLifecycle in XML. Replay=0 means new collectors don't re-fire old events; extraBufferCapacity=1 covers brief gaps.

Advanced Q127. What's a SingleLiveEvent and why is it deprecated?

An old hack to deliver one-shot events through LiveData (which always replays the latest value). It's brittle, hard to compose, doesn't handle multiple observers. Replaced by SharedFlow(replay=0) or a Channel exposed as Flow.

Advanced Q128. How do you handle a screen with multiple independent data sources?

Combine in the VM with combine(flow1, flow2, flow3) { a, b, c -> ScreenState(a, b, c) }, exposed as StateFlow. UI sees one state object. Each source can fail/refresh independently.

Advanced Q129. What's SDUI?

Server-Driven UI. Backend returns layout descriptors (modules, items); client renders generically. Lets product change the UI without an app release. Tradeoff: client must support all module types in advance; new types require app update.

Advanced Q130. How do you handle pagination in MVI?

Paging 3 emits PagingData<T> as a Flow. In Compose: vm.pager.collectAsLazyPagingItems(). State is internal to PagingData — your ViewModel exposes val pager: Flow<PagingData<Item>> = ... .cachedIn(viewModelScope). Loading/error states surfaced via loadState.

Advanced Q131. What anti-patterns should you call out in an architecture review?

(1) Fat ViewModel (extract use cases). (2) Repository returning DTOs (map to domain). (3) SingleLiveEvent (use SharedFlow). (4) Bare collectAsState (lifecycle-aware). (5) Activity holding the data (use ViewModel). (6) Singleton object Repo (use DI). (7) Multiple sources of truth (drift bugs).

Extreme Q132. Defend MVVM over MVI for a 50-screen app.

"For a 50-screen app, MVI's boilerplate per screen is high. I'd default to MVVM — single VM with multiple StateFlows — and lift to MVI only on the screens where state interactions are truly complex (player, multi-step forms). The discipline of state hoisting and unidirectional flow can be applied without the full MVI ceremony. Mixing is fine — it's a per-screen choice."

Extreme Q133. How do you architect for an app that needs to support both phone and Android TV with shared business logic?

Multi-module: :core-domain, :core-data, :core-network shared; :ui-phone, :ui-tv separate; :app-phone, :app-tv entry points. Build variants for platform-specific resources. Use internal visibility liberally. ViewModels are typically shared; only the UI layer differs.

Extreme Q134. How would you split a monolith into multi-module mid-flight (without halting feature work)?

Strangler-fig: start by extracting :core first (network, DB, design-system) — least churn. Then peel one feature at a time off into :feature:X. Use internal to enforce module-only visibility. Keep the original module as a "legacy" landing zone. Run alongside features for 2-3 quarters.

Extreme Q135. Defend Use Cases vs "VM directly calls Repo."

"Use cases shine when (1) logic is reused across multiple VMs, (2) it composes multiple repos, (3) it has non-trivial logic worth unit-testing in isolation. For simple passthrough, they're bloat. I make the choice per-feature, not as dogma. The honest test: 'does extracting this use case make any caller cleaner or any test easier?' If no — skip it."

Hilt / DI (136–155)

Basic Q136. What is dependency injection?

A pattern where a class receives its dependencies from outside (constructor/setter/field) rather than constructing them itself. Decouples implementation, improves testability.

Basic Q137. What does @HiltAndroidApp do?

Annotates the Application class. Triggers Hilt code generation for the app component (the root of the DI graph).

Basic Q138. What does @AndroidEntryPoint do?

Annotates an Activity/Fragment/Service/Receiver/View — enables field injection. The class becomes a member-injectable Hilt entry point.

Basic Q139. @Inject constructor — what does it do?

Tells Hilt how to construct this class. Hilt resolves the constructor's parameters from the graph.

Intermediate Q140. @Provides vs @Binds?

@Provides: function in an object module returning the binding — works for any expression. @Binds: abstract function in an abstract module mapping interface to impl — more efficient (Dagger doesn't generate a factory class). Use @Binds for interface→impl, @Provides for configured types.

Intermediate Q141. List Hilt's components in scope-order.

SingletonComponent → ActivityRetainedComponent → ViewModelComponent OR ActivityComponent → FragmentComponent → ViewComponent (or ViewWithFragmentComponent). Plus ServiceComponent.

Intermediate Q142. Can a @Singleton dep be injected into @ActivityScoped?

Yes. Wider scope into narrower is allowed. The reverse (@ActivityScoped into @Singleton) is forbidden — narrower into wider doesn't make sense (the dep would outlive its scope).

Intermediate Q143. What's a @Qualifier?

A custom annotation to disambiguate same-type bindings. @IoDispatcher vs @MainDispatcher for CoroutineDispatcher. Without qualifiers, Hilt would see "duplicate binding" for two bindings of the same type.

Intermediate Q144. How do you inject into a non-Hilt class?

@EntryPoint + @InstallIn — declare an interface with the bindings you need. EntryPointAccessors.fromApplication(ctx, MyEntryPoint::class.java). Used for legacy code or third-party callbacks.

Intermediate Q145. What is @HiltViewModel?

Annotates a ViewModel for Hilt to provide. In a Fragment: val vm: MyVm by viewModels(). In Compose: val vm: MyVm = hiltViewModel(). Hilt + Navigation Compose auto-wires SavedStateHandle with route args.

Advanced Q146. What is assisted injection?

For classes that need both DI'd deps and runtime args. @AssistedInject constructor(val repo: Repo, @Assisted val userId: String) + @AssistedFactory interface Factory { fun create(userId: String): MyClass }. Use when SavedStateHandle isn't appropriate.

Advanced Q147. Multibinding — @IntoSet and @IntoMap?

Collect impls into a Set or Map. @Binds @IntoSet to bind multiple AnalyticsTracker impls into one Set. @Binds @IntoMap @StringKey("home") to bind keyed Routers into a Map. Useful for plugin-style architecture.

Advanced Q148. How do you integrate Hilt with WorkManager?

@HiltWorker class MyWorker @AssistedInject constructor(@Assisted ctx: Context, @Assisted params: WorkerParameters, deps...). In Application: implement Configuration.Provider, inject HiltWorkerFactory, set in workManagerConfiguration. Disable the default WorkManager initializer in AndroidManifest.

Advanced Q149. What's @TestInstallIn?

Replaces a production module in tests. @TestInstallIn(components = [SingletonComponent::class], replaces = [RepoModule::class]). Tests bind fakes via this test module instead of the production one.

Advanced Q150. Why @Binds over @Provides for interface→impl?

@Binds is more efficient — Dagger can generate just a forwarding call instead of a factory class. Forces you to express the binding declaratively (no construction logic). For most interface→impl bindings, prefer @Binds.

Advanced Q151. What does Hilt generate at compile time?

Components for each scope you use, factory classes for each @Inject constructor, member-injection methods for @AndroidEntryPoint-annotated classes, and module-validation code. All visible in the build's generated/ directory.

Extreme Q152. Why does @Inject lateinit var x: Foo in a non-@AndroidEntryPoint Activity throw at runtime?

Hilt generates the member-injection code only for classes annotated @AndroidEntryPoint. Without the annotation, no code generation, no injection — x stays uninitialized.

Extreme Q153. Hilt vs Koin vs manual DI?

Hilt: compile-time graph (Dagger), Android-only, build-time errors. Koin: runtime DSL, KMP-friendly, runtime errors. Manual DI (constructor injection + a few wiring functions): no library, simplest, harder to scale. For Android-only large apps: Hilt. For KMP shared modules: Koin or manual.

Extreme Q154. How do you debug "MissingBinding"?

Compile error tells you which type and which graph it's missing from. (1) Check the type has @Inject constructor OR a @Provides/@Binds. (2) Check the module is @InstallIn the right component. (3) Check qualifiers match if any. (4) Check no version mismatch (KSP vs kapt confusion).

Extreme Q155. Can you have circular dependencies in Hilt?

Direct cycles fail to compile. Use dagger.Lazy<T> or Provider<T> to break the cycle by deferring resolution. Common case: TokenAuthenticator needs AuthApi which needs OkHttpClient which has the Authenticator — wrap one side in Lazy.

Jetpack Libraries (156–185)

Basic Q156. ViewModel — what survives, what doesn't?

Survives configuration change (rotation). Does NOT survive process death. SavedStateHandle survives both — Bundle-based, small data only.

Basic Q157. LiveData vs StateFlow?

LiveData: lifecycle-aware (auto-pauses off-screen). StateFlow: not lifecycle-aware on its own; needs repeatOnLifecycle or collectAsStateWithLifecycle. New code prefers StateFlow for Compose interop and Flow operators.

Basic Q158. Why Room over plain SQLite?

Compile-time SQL verification, type-safe queries, automatic Flow emission on table change, transactions via annotations, type converters, migration helpers, schema export for migration testing. Plain SQLite gives you string queries and runtime crashes when schema drifts.

Basic Q159. What is WorkManager for?

Deferrable, guaranteed background work — sync, upload, indexing. Survives app restart, device reboot. Picks the right backend internally. Use over JobScheduler/AlarmManager.

Basic Q160. DataStore vs SharedPreferences?

DataStore: async (Flow-based), transactional, type-safe (proto). SharedPreferences: synchronous (main-thread risk), no schema. Use DataStore for new code; migrate from SharedPreferences as you touch each surface.

Intermediate Q161. How do you write a Room migration?

val MIGRATION_1_2 = object : Migration(1, 2) { override fun migrate(db) = db.execSQL("ALTER TABLE...") }. Add to Room.databaseBuilder().addMigrations(MIGRATION_1_2). Set exportSchema=true and check the generated JSON into Git.

Intermediate Q162. How do you observe joined data with Room?

@Transaction + @Relation on a composed data class returned from a @Query that returns Flow. The transaction ensures a consistent snapshot.

Intermediate Q163. WorkManager constraints?

NetworkType.UNMETERED, requiresCharging, requiresBatteryNotLow, requiresStorageNotLow, requiresDeviceIdle. Combined into a Constraints object on the request.

Intermediate Q164. Expedited work?

For ASAP background work that can run as a foreground service if needed. setExpedited(OutOfQuotaPolicy.RUN_AS_NON_EXPEDITED_WORK_REQUEST) + override getForegroundInfo. User-visible importance required.

Intermediate Q165. Can WorkManager run while the app is killed?

Yes — WorkManager persists the queue to disk; the OS schedules its execution, even after process death and reboot.

Intermediate Q166. Preferences DataStore vs Proto DataStore?

Preferences: untyped key-value, like SharedPreferences but async/Flow. Proto: typed schema (protobuf), better for structured app state, type-safe at compile time. Choose proto for any non-trivial state.

Intermediate Q167. How do you migrate from SharedPreferences to DataStore?

preferencesDataStore("settings", produceMigrations = { ctx -> listOf(SharedPreferencesMigration(ctx, "old_prefs")) }). First read of DataStore migrates and deletes the old prefs file.

Intermediate Q168. How do you do type-safe Navigation in Compose?

Nav 2.8+: @Serializable data class DetailRoute(val id: String); composable<DetailRoute> { entry -> val args = entry.toRoute<DetailRoute>() }. Compile-time arg type checking.

Advanced Q170. How do you implement infinite scroll with Paging 3 + offline-first?

Pager with RemoteMediator: DB is source of truth. Mediator fetches network for next page, writes to DB. pagingSourceFactory = { db.dao().pagingSource() } reads back from DB. UI sees one Flow.

Advanced Q171. How do you handle Coil image cache sizing?

In Application : ImageLoaderFactory, configure memoryCache { maxSizePercent(0.25) } (25% of heap) and diskCache { maxSizeBytes(100MB) }. Cap memory to leave room on low-end devices.

Advanced Q172. What's RemoteMediator.LoadType?

REFRESH (initial / pull-to-refresh), APPEND (next page), PREPEND (previous, often unused for forward-only feeds). Your mediator decides what to fetch for each type.

Advanced Q173. How do you avoid main-thread DB I/O with Room?

DAOs return Flow (cold, runs upon collect; collected from a coroutine on IO dispatcher) or are suspend functions. Room rejects synchronous calls on main by default — allowMainThreadQueries() only in tests.

Advanced Q174. What's WorkManager.beginUniqueWork vs enqueueUniqueWork?

Both ensure only one chain with the given name runs at a time. beginUniqueWork returns a continuation you can chain (then(...)). enqueueUniqueWork takes a single request directly.

Extreme Q175. How does Room's invalidation tracker work?

Room maintains an internal table tracking changes. DAO Flow queries register against the tables they touch. On change, Room re-runs the query and emits a new value into the Flow. Cheaper than a manual cursor loop and lifecycle-safe.

Extreme Q176. How do you handle a Room migration that reshapes data (not just adds a column)?

(1) Create the new table with the desired shape. (2) Copy data from old → new with whatever transformation needed (SQL INSERT INTO new SELECT ... FROM old). (3) Drop the old table. (4) Rename new → old. Test with the schema export — Room verifies the resulting schema matches version N.

Extreme Q177. WorkManager backoff strategy — linear vs exponential?

Linear: delay = initial × attempt. Exponential: delay = initial × 2^(attempt-1). Exponential for transient failures (server overload). Linear for predictable retry cadence. Configure with setBackoffCriteria(BackoffPolicy.EXPONENTIAL, 30, TimeUnit.SECONDS).

Intermediate Q178. How do you save scroll position across navigation?

Compose: rememberLazyListState() survives within a screen; for cross-screen, hoist into a SavedStateHandle in VM or rememberSaveable. XML: SavedStateHandle in VM; or rely on Fragment's view state restoration.

Intermediate Q179. How do you do deep links in Navigation Compose?

composable<Route>(deepLinks = listOf(navDeepLink { uriPattern = "myapp://detail/{id}" })). App Links require domain verification (digital asset link).

Intermediate Q180. Application vs ApplicationContext?

Application is the singleton class. ApplicationContext is its Context view (application.applicationContext). Always inject @ApplicationContext Context, not raw Application — easier to fake in tests.

Intermediate Q181. Why viewLifecycleOwner in Fragments?

A Fragment can outlive its view (back stack). this as LifecycleOwner ties observers to the Fragment lifecycle (longer than the view). viewLifecycleOwner ties to the view, so observers stop when the view dies. Always use viewLifecycleOwner for UI-bound observers.

Advanced Q182. How do you observe navigation results going BACK (returning data to the previous screen)?

Compose: use the previous entry's SavedStateHandle: navController.previousBackStackEntry?.savedStateHandle?.set("key", value); on the previous screen, observe via currentBackStackEntry?.savedStateHandle?.getStateFlow("key", default).

Advanced Q183. WorkManager.ExistingWorkPolicy — KEEP / REPLACE / APPEND?

KEEP: if work with this name is queued/running, no-op. REPLACE: cancel existing and enqueue new. APPEND: chain after existing. APPEND_OR_REPLACE: append if any pending, replace if all completed.

Advanced Q184. How do you cap a Coil image's memory cost?

Set the request's target size with .size(Size(width, height)) — Coil decodes at that size, not the source's. Use Modifier.size in Compose; AsyncImage respects layout constraints. Critical for grid views with many small images.

Extreme Q185. How would you implement a sync engine that handles offline edits + conflicts?

Each entity has id (client UUID), serverId, version, status (PENDING_CREATE/UPDATE/DELETE/SYNCED). SyncWorker walks pending → POST/PATCH/DELETE → on 409 conflict, run resolution (last-write-wins / user prompt / merge). Pull deltas via since timestamp. Mark synced; reset versions.

Networking & Auth (186–205)

Basic Q186. What is REST?

An architectural style for HTTP APIs. Resources identified by URLs; operations via standard verbs (GET/POST/PUT/PATCH/DELETE); stateless (each request stands alone). Status codes signal outcome (2xx success, 4xx client error, 5xx server error).

Basic Q187. What does Retrofit do?

Converts a Kotlin interface annotated with HTTP method annotations into a typed HTTP client at runtime. Handles serialization via converter factories.

Basic Q188. What is OkHttp?

The HTTP client Retrofit sits on top of. Handles connection pooling, gzip, HTTP/2, redirects, caching, interceptors.

Basic Q189. What's GSON?

Google's JSON library — converts JSON ↔ Java/Kotlin objects via reflection. Common in legacy code; for new Kotlin code prefer kotlinx.serialization or Moshi (compile-time codegen, faster, Kotlin-aware).

Intermediate Q190. Why kotlinx.serialization over Gson?

Compile-time codegen (KSP) — faster cold start, no reflection. Kotlin-aware: handles default args, nullability, sealed classes, value classes natively. Multiplatform-ready. Smaller APK (no reflection lib).

Intermediate Q191. How do you add an Authorization header to every request?

OkHttp Interceptor — append Authorization: Bearer $token to the request before forwarding: chain.proceed(chain.request().newBuilder().header(...).build()).

Intermediate Q192. Interceptor vs Authenticator?

Interceptor: runs on every request — add headers, logging, retry. Authenticator: runs on 401 — refresh token + retry the original request. Both apply at the OkHttpClient level.

Intermediate Q193. How do you log requests/responses?

HttpLoggingInterceptor with level = BODY in debug, NONE in release. Never log BODY in release — leaks PII and tokens.

Intermediate Q194. What is OAuth 2.0 PKCE?

Proof Key for Code Exchange — the mobile-safe OAuth flavor. Mobile apps can't safely store a client secret; PKCE replaces it with a per-flow random verifier. App generates verifier, sends SHA256 hash as code_challenge, exchanges code + verifier for tokens.

Intermediate Q195. Where do you store OAuth tokens?

EncryptedSharedPreferences (Jetpack Security) — backed by Android Keystore so the encryption key is hardware-isolated. Never plain SharedPreferences or files.

Intermediate Q196. What's certificate pinning?

Hardcoding the expected server cert hash so a rogue CA can't MITM. CertificatePinner on OkHttpClient or via network_security_config.xml. Pin to multiple keys (current + backup) — single-key pin can brick the app on cert rotation.

Advanced Q197. How do you handle concurrent 401s during token refresh?

Wrap the refresh in a Mutex.withLock. After acquiring, compare current token against the expired one — if different, another caller already refreshed; just retry with the new token. If same, do the refresh.

Advanced Q198. Why use idempotency keys?

POST is not naturally idempotent — retrying after a network drop can create duplicates. Send Idempotency-Key: <UUID>; server dedupes. The same key MUST be reused on retry (otherwise dedup fails).

Advanced Q199. Cursor vs page-number pagination — which and why?

Cursor: stable under concurrent inserts/deletes (server returns nextCursor; client sends it back). Page-number: simple but unstable — a new item inserted between pages causes "skip" or "duplicate" rows. Prefer cursor for production.

Advanced Q200. How do you secure WebView?

Disable setAllowFileAccess, setAllowFileAccessFromFileURLs, setAllowUniversalAccessFromFileURLs. Avoid @JavascriptInterface unless absolutely needed (annotated method opens an attack surface). Validate redirected URLs against an allowlist. Use WebViewAssetLoader for serving local content.

Advanced Q201. How would you implement an OkHttp interceptor that retries 5xx with exponential backoff? 
class RetryInterceptor(private val maxRetries: Int = 3) : Interceptor {
    override fun intercept(chain: Chain): Response {
        var attempt = 0
        var response: Response
        while (true) {
            response = chain.proceed(chain.request())
            if (response.code !in 500..599 || attempt >= maxRetries) return response
            response.close()
            attempt++
            Thread.sleep((1L shl (attempt - 1)) * 500L)   // 500, 1000, 2000ms
        }
    }
}
Extreme Q202. Walk through the full PKCE flow.

(1) App generates code_verifier (random 43-128 chars). (2) Computes code_challenge = BASE64URL(SHA256(verifier)). (3) Opens Chrome Custom Tab → auth URL with code_challenge, code_challenge_method=S256, state. (4) User logs in → server redirects with code + state. (5) App validates state matches, POSTs {code, code_verifier, redirect_uri, client_id} to /token. (6) Server validates SHA256(verifier) == challenge, returns access + refresh tokens. (7) App stores in EncryptedSharedPreferences.

Extreme Q203. What's refresh token rotation?

Each /refresh call returns BOTH a new access token AND a new refresh token. The old refresh token is revoked server-side. If the old one is used (or seen replayed by an attacker), server detects compromise and invalidates the family — forcing re-auth. Mitigates refresh token theft.

Extreme Q204. How does HTTP/2 differ from HTTP/1.1 for mobile apps?

HTTP/2: multiplexes multiple requests over a single TCP connection (no more per-domain connection limits), header compression (HPACK), server push (rare on mobile). For Android/OkHttp: free benefit on supported servers — fewer connection re-establishments, lower latency on mobile networks.

Extreme Q205. How do you secure a deep link that takes a userId?

Validate the userId server-side against the current session (the link could be shared). Use App Links (with digital asset link verification) so only your app handles your domain — prevents another app intercepting. Treat all URL params as untrusted user input.

Android SDK Core (206–225)

Basic Q206. List the Activity lifecycle methods in order.

onCreate → onStart → onResume → (interactive) → onPause → onStop → onDestroy. From stopped: onRestart → onStart → onResume.

Basic Q207. What's the Fragment lifecycle?

onAttach → onCreate → onCreateView → onViewCreated → onStart → onResume → onPause → onStop → onDestroyView → onDestroy → onDetach.

Basic Q208. What is an Intent?

A messaging object — a request to start an activity, deliver a broadcast, or start a service. Explicit (specifies the target class) or implicit (system picks the handler).

Basic Q209. What are the four core Android components?

Activity (UI screen), Service (background work), BroadcastReceiver (event listener), ContentProvider (cross-app data).

Intermediate Q210. Configuration change vs process death — what survives?

Config change (rotation): ViewModel survives; onSaveInstanceState Bundle survives. Process death: only the Bundle / SavedStateHandle survives — ViewModel is recreated.

Intermediate Q211. Standard launch mode vs singleTop?

Standard: always create a new instance (A→A→A allowed). singleTop: if activity already on top, reuse + fire onNewIntent; otherwise create new. Use singleTop for screens reachable via notification to avoid stacking duplicates.

Intermediate Q212. singleTask vs singleInstance?

singleTask: one instance across the task; if exists anywhere in the stack, bring to top + clear above. singleInstance: lives alone in its own task — no other activity in this task. singleInstance is rare (system dialogs); singleTask for "Home" / launcher entry.

Intermediate Q213. App opens A → B → C. Notification opens A with singleTask. Stack now?

Stack: just A. singleTask brings A to top and clears B, C. onNewIntent fires on the existing A.

Intermediate Q214. FLAG_ACTIVITY_CLEAR_TOP — what does it do?

Pops every activity above the existing instance of the target activity. Combined with FLAG_ACTIVITY_NEW_TASK + FLAG_ACTIVITY_CLEAR_TASK for "go home and clear stack."

Intermediate Q215. Foreground service vs background service vs bound service?

Foreground: persistent notification, OS keeps process alive (music, location). Background: hidden, killed eagerly on Android 8+ (use WorkManager instead). Bound: lives only while clients are bound (IPC).

Advanced Q216. Why are static BroadcastReceivers restricted on Android 8+?

Implicit broadcasts can wake many apps simultaneously, draining battery. Android 8+ limits which implicit broadcasts can be statically registered. Use dynamic registration with explicit lifecycle, or migrate to alternative mechanisms (NetworkCallback, ProcessLifecycleOwner, FCM).

Advanced Q217. What's FLAG_IMMUTABLE on PendingIntent and why required on Android 12+?

Immutable PendingIntent prevents another app from modifying the wrapped intent (e.g., changing the target component or adding extras). API 31+ requires you to explicitly choose IMMUTABLE or MUTABLE — used to be implicit. MUTABLE only when truly needed (e.g., inline notification reply where the system fills in the result).

Advanced Q218. How does Doze mode affect your app?

Device unplugged + screen off + stationary → OS defers background CPU/network. Wakes briefly periodically. Foreground services and high-priority FCM bypass. Implication: WorkManager handles this; avoid setExactAndAllowWhileIdle alarms unless you need wall-clock precision.

Advanced Q219. What is App Standby?

OS bucketing of apps based on usage frequency: active / working set / frequent / rare / restricted. Lower buckets get tighter restrictions on jobs, alarms, FCM. Implication: don't expect frequent background execution if user rarely opens your app.

Advanced Q220. onSaveInstanceState — when does it run?

Before onStop when the OS might destroy the activity (rotation, going to background where memory might be reclaimed). NOT called when user explicitly finishes the activity (back press, finish()). Stored in a Bundle restored to onCreate / onRestoreInstanceState if recreated.

Advanced Q221. How would you structure init order in Application.onCreate?

Light only: required SDK config that must run before first activity. Defer everything else via App Startup library or post-first-frame coroutine. Crashlytics — install early. Analytics SDK — initialize lazily on first event. DI graph — Hilt is lazy by default.

Extreme Q222. Trace what happens when user rotates the device on a Compose screen.

Activity is destroyed (onPause→onStop→onDestroy). New Activity created (onCreate). setContent { } runs again. ViewModel survives (stored in ActivityRetainedComponent). rememberSaveable values restored from Bundle. remember values lost (re-initialized). LaunchedEffect with the same keys runs again.

Extreme Q223. What's the difference between android:configChanges and not declaring it?

Without it: Activity is recreated on the listed config changes. With configChanges="orientation|screenSize": onConfigurationChanged is called instead — the Activity persists. Use sparingly — defeats lifecycle assumptions; can hide bugs in state restoration.

Extreme Q224. Difference between Application Context and Activity Context?

Activity Context: tied to Activity lifecycle, has theme + window. Application Context: app lifetime, no theme/window. Use Application context for long-lived references (singletons, repos) to avoid leaking the Activity. Use Activity context for UI work (dialogs, themed inflation).

Extreme Q225. How do you migrate a Service-heavy app to WorkManager?

Audit each Service: (1) If user-visible long work — keep as foreground service (with foregroundServiceType). (2) If deferrable background — replace with WorkManager request. (3) If one-shot post-event — replace with OneTimeWorkRequest. (4) If bound service for IPC — keep. Test on Android 8+ to verify background restrictions don't break flows.

Performance & Profiling (226–245)

Basic Q226. Cold / warm / hot start?

Cold: process newly created (Application.onCreate runs). Warm: process exists, Activity recreated. Hot: process + Activity exist, just brought to foreground.

Basic Q227. How do you measure cold start?

adb shell am start -W -n pkg/.MainActivity for quick. Macrobenchmark StartupBenchmark for stable, CI-friendly numbers. Profiler "System Trace" for one-time deep dive.

Basic Q228. Frame budget at 60Hz?

16.6ms. At 90Hz: 11.1ms. At 120Hz: 8.3ms. Anything past budget → dropped frame → user-perceptible jank.

Intermediate Q229. What are Baseline Profiles?

AOT compile hints shipped with the APK. ART pre-compiles those code paths so they don't pay JIT cost on first execution. Generated via Macrobenchmark. Often 20-40% cold start improvement.

Intermediate Q230. Top tactics for cold start?

(1) Baseline Profiles + R8. (2) Defer SDK init via App Startup library. (3) Trim Application.onCreate. (4) Lazy DI graph (Hilt is by default). (5) Don't block main on disk/network in onCreate. (6) SplashScreen API correctly.

Intermediate Q231. Common jank sources?

Heavy work on main (JSON parsing, large bitmap decode), Compose unstable parameters, RecyclerView onBindViewHolder doing I/O, overdraw, bitmap loading without sampling.

Intermediate Q232. What is JankStats?

An AndroidX library for runtime jank detection in production-like builds. Reports per-frame data with metadata you set. Useful for collecting jank metrics from real users.

Intermediate Q233. What is StrictMode?

Debug-only enforcement of best practices. ThreadPolicy: detect disk reads/writes/network on main thread. VmPolicy: detect leaked Closeables, leaked Activities, etc. Catches bugs before they ANR or leak in production.

Intermediate Q234. Common memory leak sources?

Singletons holding Activity/Context, anonymous inner classes, listeners not unregistered, GlobalScope.launch capturing Activity, ExoPlayer not released, RxJava subscriptions not disposed (legacy), unbounded bitmap caches.

Intermediate Q235. What does R8 do?

Code shrinking + obfuscation + optimization. Default in release with minifyEnabled true. Combined with shrinkResources true for APK size reduction. Requires ProGuard rules for reflection-heavy code.

Advanced Q236. ANR triggers?

UI thread blocked > 5s for input. BroadcastReceiver onReceive > 10s. Service onStart/onStartCommand > 20s. Look at main thread stack at the moment of ANR (Play Console Vitals or /data/anr/traces.txt on rooted devices).

Advanced Q237. APK size — top reductions?

R8 + resource shrinking, App Bundles (per-device APKs), WebP + vector drawables, dynamic feature modules, audit large transitives, language-specific splits via Bundles.

Advanced Q238. Compose-specific perf knobs?

Stable parameters (@Immutable), derivedStateOf for derived booleans, lazy list key+contentType, function references over inline lambdas, Compose Compiler reports in CI.

Advanced Q239. How do you track production performance?

Play Console Vitals (ANR, crash rate, slow rendering, slow start). JankStats library for richer per-frame data. Crashlytics for crash trends. Custom: Conviva for player health, Dynatrace/NewRelic for app traces.

Advanced Q240. What's TTID vs TTFD?

Time to Initial Display: first frame drawn. Time to Full Display: content fully populated. Mark TTFD with reportFullyDrawn() from your Activity/Compose. Macrobenchmark distinguishes them.

Advanced Q241. How do you debug an OOM in production?

Reproduce locally → Profiler heap dump at the point of OOM. Group instances by class — anything >100MB, anything growing unboundedly. Common: bitmap caches, String pools, leaked Activities. Verify fix with a 7-day Crashlytics window post-release.

Extreme Q242. How do Baseline Profiles work under the hood?

The profile is a binary list of method signatures + flags (hot, startup, post-startup). At install time, ART compiles those methods AOT (ahead of time), generating optimized native code. On first execution, those methods skip JIT cost. The profile ships in baseline.prof within the APK.

Extreme Q243. How would you reduce startup time on a 10-screen app?

Steps in order: (1) Macrobenchmark to establish baseline. (2) Generate Baseline Profiles for the cold-start path. (3) Audit Application.onCreate — defer all SDK init via App Startup. (4) Migrate to kotlinx.serialization if Gson is on the cold path. (5) Network: cache + prefetch hot manifests. (6) Compose: stability annotations, eliminate non-skippable parents. (7) R8 + resource shrinking. Verify each step with Macrobenchmark.

Extreme Q244. What's the difference between heap allocation and JIT in terms of startup cost?

Heap allocations are fast in steady state but contribute to GC pressure during startup if you allocate aggressively. JIT compilation costs CPU on first execution of a method. Baseline Profiles target the JIT cost (AOT-compile critical paths). For allocations, look at Application.onCreate for any avoidable churn.

Extreme Q245. How do you regression-test performance in CI?

Macrobenchmark module with StartupBenchmark + targeted scroll/render benchmarks. Run on a fixed device (or device farm) on each commit. Compare numbers vs baseline; alert on >10% regression. Compose Compiler reports also CI-checked.

Video Player (246–260)

Basic Q246. Why ExoPlayer over MediaPlayer?

ExoPlayer: open-source, customizable, consistent across OS versions, supports HLS/DASH/SmoothStreaming/DRM out of the box, handles ABR. MediaPlayer: OS-tied, limited customization, behavior varies by OEM, no full ABR.

Basic Q247. HLS vs DASH?

HLS: Apple, .m3u8 manifest, .ts/.mp4 segments, native iOS support. DASH: MPEG, .mpd XML, .mp4 segments, no native iOS. Modern: HLS+CMAF works almost everywhere — single packaging serves both.

Intermediate Q248. What is ABR?

Adaptive Bitrate. Player measures bandwidth on segment downloads and buffer health, picks a bitrate from the manifest's variants. Switches up if buffer healthy + bandwidth allows; down if buffer drains.

Intermediate Q249. Widevine DRM levels?

L1: hardware-backed (TEE) — most secure, allows HD/4K. L2: mixed. L3: software only — typically SD; HD often blocked by content owners.

Intermediate Q250. CSAI vs SSAI?

CSAI: client fetches ad and pauses content during break — separate fetch, vulnerable to ad blockers, easy per-user personalization. SSAI: server stitches ads into the stream — single continuous playback, no buffer at break, harder ad-block, harder personalization (per-user manifests).

Intermediate Q251. What are ad quartile beacons?

HTTP fire-and-forget pings to ad partners at lifecycle moments: impression/start, firstQuartile (25%), midpoint (50%), thirdQuartile (75%), complete (100%), plus click/error/skip. Servers use these for billing and measurement.

Advanced Q252. How do you do offline DRM playback?

(1) Download stream segments via DownloadManager. (2) Pre-fetch and persist the Widevine license via OfflineLicenseHelper. (3) On playback, hand the license keyId to player via DrmConfiguration.setKeySetId().

Advanced Q253. How would you architect a player module?

Wrap ExoPlayer in PlayerController interface. Inject via Hilt. Expose StateFlow<PlaybackState> driven by Player.Listener callbacks. Compose UI via AndroidView { PlayerView } + DisposableEffect for release. Decouples business logic and tests from the SDK; easy to swap.

Advanced Q254. What player health metrics do you track?

TTFF (time to first frame), rebuffer ratio (rebuffer time / total play time), bitrate distribution, error count by type (DRM/network/decoder). Report to Conviva or Bitmovin Analytics, plus internal events.

Advanced Q255. How do you handle a 4G→Wifi mid-playback?

ExoPlayer recovers automatically — segments fail, retry, continue. ABR re-evaluates bandwidth on the new connection. If retries exhaust, surface a recoverable error and call player.prepare() again.

Extreme Q256. How do you do PiP with ExoPlayer?

Manifest: android:supportsPictureInPicture="true" on the activity. configChanges="screenSize|smallestScreenSize|screenLayout|orientation". Call enterPictureInPictureMode(params). Override onPictureInPictureModeChanged to hide/show controls. Player keeps playing.

Extreme Q257. What's MediaSession and why use it?

Exposes playback state to the system: lock screen, Bluetooth headsets, Wear, Auto, Cast. Single source of truth for playback. Required if you want media controls outside the app. Media3's MediaSessionService is the modern API.

Extreme Q258. How would you A/B test players?

Behind a feature flag, swap the PlayerController impl. Cohort A gets ExoPlayer; cohort B gets Bitmovin. Compare TTFF, rebuffer ratio, error rate. Roll forward or back based on metrics. Important: ad lifecycle and analytics must work with both impls — wire through PlayerController.

Extreme Q259. How would you debug a "DRM license error 6004"?

Look up code (varies — usually a license server response). Check: (1) device security level (L1 vs L3 mismatch with content). (2) Network — is the license URL reachable? (3) Auth headers — is your token valid? (4) Provisioning — has the device been provisioned by Widevine ProvisioningServer? (5) Time — device clock skew can break DRM.

Extreme Q260. SSAI vs CSAI — when should I choose which for a streaming product?

SSAI for live + premium content where ad-block circumvention matters and seamless playback is critical. CSAI when you need fine-grained per-impression targeting that's hard to do server-side, or when your ad backend isn't SSAI-capable. Hybrid is common: SSAI for video pre-rolls, CSAI for display/banner ads in the UI.

System Design + Behavioral (261–280)

Intermediate Q261. How do you start an HLD round?

Clarify scope first (5-8 minutes — never skip): scale, online/offline, real-time, platforms, auth. Then list functional + non-functional reqs. Sketch architecture. Pick a deep-dive area. Articulate tradeoffs throughout.

Intermediate Q262. Design a Twitter-like feed.

Cursor-pagination (stable). Image prefetching (next 3 items). Cache: LRU for recent ~500 posts. Optimistic likes. Pull-to-refresh: re-fetch from cursor=null, prepend. Read receipts: count when item visible >2s + >50% on screen.

Intermediate Q263. Design an offline-first notes app.

Room is source of truth. Each entity has clientUUID + serverId + version + status. WorkManager SyncWorker drains pending creates/updates/deletes. Conflict: last-write-wins or 409+prompt. UI observes Flow from DAO.

Advanced Q264. Design an image loader.

Memory cache (LruCache, ~1/8 heap) → disk cache (LRU on disk) → network (OkHttp). Cache key = url + size + transformations. Cancel on view recycle (track requests by target). Decode on Default dispatcher. Bind on Main.

Advanced Q265. Design a chat client.

WebSocket inbound. Room for persistence. Send queue with WorkManager + client UUIDs (idempotency). Reconnect with exponential backoff + "since lastDeliveredId" replay. Pagination via keyset on messageId DESC.

Advanced Q266. Design analytics integration usable from both Compose and XML.

AnalyticsTracker interface. Hilt multibinds multiple impls (NewRelic, Crashlytics, internal). AnalyticsHub fans out, off-main, swallows failures. Compose: CompositionLocal. XML: base Fragment with onResume firing screen event.

Advanced Q267. Design push notifications end-to-end.

FCM token registered on app start; server side subscribes to topics or stores device tokens. Notification channels (Android 8+) created at app start. Deep links via TaskStackBuilder. Silent push for sync. Privacy: redact on lockscreen via setVisibility.

Advanced Q268. Design a photo upload pipeline.

WorkManager foreground service (user-visible long work). Resumable upload via chunked PUT (S3 multipart or Tus). Persist chunk progress so reboot resumes. Notification with progress. Constraints: UNMETERED if user toggles wifi-only.

Advanced Q269. Design a video streaming app like Tubi.

See system-design page section 3. Key parts: SDUI for home rows, ExoPlayer/Media3 + Widevine + HLS+CMAF + SSAI, beacon tracking, continue-watching with offline-first DB, PKCE auth, custom analytics ingestion, multi-module structure.

Extreme Q270. Design a feature flag system.

Local cache (DataStore) — synchronous read at app start. Remote (Firebase Remote Config or LaunchDarkly). Refresh policy: on app start + every 12h. QA override via developer settings. Type-safe accessors per flag. Fallback to default if remote fetch fails.

Extreme Q271. Two devices edit the same note offline. How do you reconcile?

Three options to articulate: (1) Last-write-wins (simple, lossy). (2) Version-based 409 + user prompt (better UX). (3) Three-way merge (Notion/Figma-level, complex). Pick one per product and acknowledge alternatives. Server-generated content hash dedupes identical edits.

Extreme Q272. How would you handle a 10× user-count spike?

Backend scaling is primary (CDN, caching layer, sharding). Mobile-side: more aggressive caching, batched API calls (or GraphQL with sparse fields), feature flags for risky rollouts, robust telemetry, A/B test infra, baseline profiles for cold start at p95.
